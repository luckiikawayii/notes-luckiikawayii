The grid

The grid 是一个最大宽度为960px的12列能移动的网格，随着浏览器或设备尺寸减小而缩小，最大宽度可以用一行css改变，并且所有的列都将相应的调整大小，语法很简单并且它更容易 迅速反应编码，开始调整浏览器

Typography

类型都是用设置的，所以字体大小和空间关系可以以 <html> 为基础字体大小来响应大小，开箱即用，skeleton从不改变 <html> 字体大小，万一你需要在你的项目上面用到它，所有的尺寸都是以10为基础，code><h1> 以5.0rem字体大小就意味着50px

typography基础 Raleway 由google提供，设置在15rem (15px)到1.6 行高（24px)，其他类型的像anchors,strong, emphasis, 和 underline，显然的都包含在内

Headings创建一个大小不同的系列，每个家族都有特定的letter-spacing,line-height,和margins.

Buttons

在skeleton中，buttons有两种基本样式，标准的<button> 样式是普通的，而.button-primary按钮则是充满活力和突出的， 按钮样式应用于许多适当的表单元素，但也可以使用.button类任意附加到锚点

Forms

forms是一个巨大的麻烦，但希望这些样式使它更容易一点，所有输入选择和按钮都被规范为跨浏览器的通用高度，因此输入可以互相堆叠或放置

Lists

  • 无序列表有基本样式
  • 它们使用圆列表样式
    • 感觉设计成嵌套样式列表
    • 可以把任意一种类型的列表嵌套到另一种
  • 只是更多的列表项目mama san
  1. 有序列表也有基本样式
  2. 它们使用十进制列表样式
    • 有序和无序可以嵌套
    • 可以将任意一种列表嵌套到另一种
  3. 最后一个列表项目只是兴趣

Media queries

skeleton使用媒体查询来服务它的可伸缩网格，但也有一个查询列表，以方便设备对站点进行样式话，这些查询是移动优先的，这意味着它们的目标是min-width， 移动优先查询是如何构建的，这是组织css的首选方法，这意味着查询之外的所有样式都适用于所有设备，然后针对更大的设备进行增强，这可以防治小型设备解析大量未使用的css， 查询大小如下

  • 桌面高清: 1200px
  • 桌面: 1000px
  • 平板电脑: 750px
  • 平板手机: 550px
  • 手机: 400px
